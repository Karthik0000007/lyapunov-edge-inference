Why this causes CI to fail
- If the runner's installed PyTorch does not accept weights_only, torch.load(...) will raise something like:
  TypeError: load() got an unexpected keyword argument 'weights_only'
- The added tests exercise checkpoint load/save code paths, so the failure will show up immediately when pytest imports/executes those tests.

Two straightforward ways to resolve it

Option A — Make code robust across PyTorch versions (recommended if you want compatibility)
- Add a small compatibility wrapper that tries torch.load with weights_only and falls back to a call without that kwarg if it's unsupported.
- Replace direct torch.load(...) calls in the repo with the wrapper.

```python name=src/torch_compat.py
from pathlib import Path
from typing import Any, Optional
import torch

def torch_load_compat(path: Path | str, map_location: Optional[str | torch.device] = None) -> Any:
    """Load a file with torch.load, supporting older/newer torch signatures.

    Tries to call torch.load(..., weights_only=True) first (for torch versions
    that accept it). If that raises a TypeError due to an unexpected kwarg,
    falls back to calling torch.load(...) without weights_only.
    """
    kwargs = {"map_location": map_location}
    try:
        # Newer torch may support weights_only; try that first.
        return torch.load(path, weights_only=True, **{k: v for k, v in kwargs.items() if v is not None})
    except TypeError:
        # Fallback for older torch where weights_only isn't accepted.
        return torch.load(path, **{k: v for k, v in kwargs.items() if v is not None})
```
